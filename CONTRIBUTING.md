# Contributing to Strata RFCs

Thank you for your interest in contributing to the Strata Language RFCs!

## Repository

This repository contains Request for Comments (RFCs) for the Strata programming language. RFCs document language features, design decisions, and implementation details.

## How to Contribute

### Proposing a New RFC

1. **Check existing RFCs**: Make sure your proposal doesn't duplicate an existing RFC
2. **Create a new RFC file**: Use the format `NNNN-feature-name.md` where NNNN is the next available number
3. **Follow the RFC template**: Include all required sections:
   - Summary
   - Status (In Progress, Implemented, Proposed, Deprecated)
   - Author (Donald Pakkies)
   - Created (date in DD/MM/YYYY format)
   - Motivation
   - Design
   - Examples
   - Implementation Details
   - Alternatives Considered
   - References

### Updating Existing RFCs

1. **Fork the repository**
2. **Create a branch** for your changes
3. **Make your changes**: Update the RFC with new information
4. **Submit a pull request**: Include a clear description of your changes

### RFC Status

- **Proposed**: Feature is proposed but not yet implemented
- **In Progress**: Feature is being implemented
- **Implemented**: Feature is fully implemented and available
- **Deprecated**: Feature is deprecated and should not be used

## Code of Conduct

- Be respectful and constructive
- Focus on technical merit
- Provide clear examples and use cases
- Document alternatives considered
